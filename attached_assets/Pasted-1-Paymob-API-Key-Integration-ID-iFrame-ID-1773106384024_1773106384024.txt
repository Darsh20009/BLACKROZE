1. تكامل بوابة دفع Paymob

    أضفت حقول الإعداد في واجهة الأدمن (API Key، Integration ID، iFrame ID، Wallet Integration ID، HMAC Secret)
    أنشأت مسار /api/payments/init — تسلسل المصادقة الثلاثي مع Paymob (auth → order → payment_key → iframe)
    أضفت مسار /api/payment-gateway/test للاختبار
    مسارات الـ callback والـ webhook مع التحقق من HMAC-SHA512
    أضفت حقل paymentGateway إلى مخطط businessConfigSchema في قاعدة البيانات

2. إصلاح 3 أخطاء في Paymob بعد التكامل

    isOnlinePaymentMethod صار يشمل paymob-card وpaymob-wallet
    صفحة checkout تستخرج معاملات URL الخاصة بـ Paymob
    مسار /api/payments/verify يتعامل مع Paymob مع fallback للتحقق عبر API

3. إضافة مسارات ناقصة

    أضفت مسار /api/pos/status لإيقاف أخطاء 404 كل 30 ثانية
    أضفت مسار /api/zatca/settings

4. إصلاح رسالة تأكيد الدفع للكاشير

    صلحت رسالة التأكيد التي تظهر للكاشير عند إتمام الدفع

5. إضافة Pagination لطلبات لوحة المدير

    أضفت عرض 20 طلب + زر "عرض المزيد" في لوحة تحكم المدير

6. إصلاح الفاتورة — عرض اسمي المنتج (عربي وإنجليزي)

    checkout.tsx: صار يحفظ nameEn مع nameAr في بيانات الطلب
    receipt-invoice.tsx: الفاتورة المعروضة والمطبوعة تظهر الاسم العربي + الإنجليزي تحته لكل منتج
    ورقة المطبخ (Kitchen Slip) صارت تعرض الاسمين أيضاً

7. تحويل شريط الأقسام في نقاط البيع

    بدّلت الشريط الجانبي الرأسي الضيق (يقطع الأسماء) إلى شريط أفقي علوي
    كل قسم يظهر بأيقونة + اسم كامل بدون قطع
    قابل للتمرير أفقياً إذا كانت الأقسام كثيرة
— عدّاد طلبات اليوم:
كان يعدّ المُلغاة ضمن عدد الطلبات اليومية. الآن يعدّ فقط الطلبات الفعلية غير المُلغاة.

مشكلة 2 — totalSpent في بطاقة الولاء:
عند إنشاء الطلب كان النظام يزيد مبلغه في رصيد مشتريات العميل، لكن عند الإلغاء لم يكن يخصمه. الآن يُعاد الخصم تلقائياً في 3 حالات:

    إلغاء العميل نفسه للطلب
    إلغاء الموظف للطلب (PATCH)
    إلغاء الكاشير للطلب (PUT)

١. لوحة تحليلات متقدمة

    أكثر المنتجات مبيعاً مع رسوم بيانية

    مقارنة الإيرادات أسبوع/شهر بشهر

٣. جدولة الموظفين والورديات
